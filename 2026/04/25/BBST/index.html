Data Structure2:Balanced Binary Search Tree

Binary Search Tree

二叉搜索树是一种查找、插入、删除在平均情况下均为 O(log⁡n)O(\log n)O(logn) 的数据结构。它满足如下性质：

1.左子树所有节点的值都小于当前节点的值。

2.右子树所有节点的值都大于当前节点的值。

BST的所有操作都由此引出。在理想情况下，插入、删除等操作正比于树的深度，因此理想复杂度为 O(log⁡n)O(\log n)O(logn)。

事实上，二叉搜索树的效率取决于insert操作是否随机，如果insert操作按照有序插入，那么二叉搜索树将很有可能退化成一条链表，在链表情况下二叉搜索树将达到最坏时间复杂度 O(n)O(n)O(n)。因此引出下面几种平衡树对二叉搜索树的改进。

AVL Tree

在AVL树中，每一个节点都有一个隐藏属性叫平衡因子(Balance Factor)。且有 BF=左子树高度−右子树高度BF = \text{左子树高度} - \text{右子树高度}BF=左子树高度−右子树高度。 可以发现BF可以衡量树倾斜的程度。

AVL树满足如下性质：

1.BF∈{−1,0,1}BF \in \{-1, 0, 1\}BF∈{−1,0,1}

2.如果 T 是一棵 AVL 树，那么其左右子树也是 AVL 树

3.树高为 O(log⁡n)O(\log n)O(logn)

关于树高性质的证明可参考：https://oi.wiki/ds/avl/

如果某个节点的 BFBFBF 变成了 222 或 −2-2−2，这棵树就倾斜了，必须通过Rotation来修复。

AVL树有如下四种Rotation情况：

1.LL 型：失去平衡的节点的左孩子的左边太重了。

操作：把失去平衡的节点的左孩子“拎”起来，让原来的父节点变成它的右孩子。

2.RR 型：失去平衡的节点的右孩子的右边太重了。

操作：想象把失去平衡的节点的右孩子“拎”起来，让原来的父节点变成它的左孩子。

3.LR 型：失去平衡的节点的左孩子的右边太重了。

操作： 先对左孩子进行一次左旋，把它变成 LL 型，然后再进行一次右旋。

4.RL型：失去平衡的节点的右孩子的左边太重了。

操作：先对右孩子进行一次右旋，把它变成 RR 型，然后再进行一次左旋。

对于每次插入元素，我们上溯到第一个失去平衡的节点，进行旋转即可。
