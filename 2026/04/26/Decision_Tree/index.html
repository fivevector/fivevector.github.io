统计学习方法 Notes: Decision Tree

Decision Tree and Feature Selection

Definition1.1 : A decision tree is a hierarchical, tree-structured predictive model that maps observations about an item to conclusions about its target value. In a decision tree:

Internal nodes represent tests on input features (attributes).

Leaf nodes hold a class label in classification tasks, or a continuous value in regression tasks.

we can note that decision tree is a set of if-then rules.

When constructing a decision tree top‑down, the core sub‑problem at every internal node is: which attribute should be used to split the data?
Our goal is to maximize the “purity” of the resulting subsets, so that each child node is dominated by a single class.

In information theory, Entropy is a measure of the uncertainty of a random variable.

Definition1.2 :Let XXX be a discrete random variable with a finite set of values {x1,x2,…,xn}\{x_1, x_2, \dots, x_n\}{x1​,x2​,…,xn​}, and its probability distribution be P(X=xi)=pi,i=1,2,…,nP(X = x_i) = p_i, i = 1, 2, \dots, nP(X=xi​)=pi​,i=1,2,…,n.

The Entropy of XXX is:

H(X)=−∑i=1npilog⁡piH(X) = -\sum_{i=1}^{n} p_i \log p_i H(X)=−i=1∑n​pi​logpi​

In the context of a dataset DDD, let ∣D∣|D|∣D∣ be the number of samples and CkC_kCk​ (for k=1,2,…,Kk=1, 2, \dots, Kk=1,2,…,K) be the classes. The entropy of dataset DDD is:

H(D)=−∑k=1K∣Ck∣∣D∣log⁡2∣Ck∣∣D∣H(D) = -\sum_{k=1}^{K} \frac{|C_k|}{|D|} \log_2 \frac{|C_k|}{|D|} H(D)=−k=1∑K​∣D∣∣Ck​∣​log2​∣D∣∣Ck​∣​

From the definition, we can see that if H(D)=0H(D) = 0H(D)=0, the dataset is perfectly “pure” (all samples belong to one class).

As we all know, When we add a feature AAA to the system, the dataset DDD will be less uncertain or “mixed”. Then we can introduce the Conditional Entropy.

Definition1.3 :H(D∣A)H(D|A)H(D∣A) measures the uncertainty of dataset DDD given the knowledge of feature AAA.

Suppose feature AAA has nnn distinct values {a1,a2,…,an}\{a_1, a_2, \dots, a_n\}{a1​,a2​,…,an​}. These values partition the dataset DDD into nnn subsets {D1,D2,…,Dn}\{D_1, D_2, \dots, D_n\}{D1​,D2​,…,Dn​}. The conditional entropy is the weighted average of the entropies of these subsets:

H(D∣A)=∑i=1n∣Di∣∣D∣H(Di)H(D|A) = \sum_{i=1}^{n} \frac{|D_i|}{|D|} H(D_i) H(D∣A)=i=1∑n​∣D∣∣Di​∣​H(Di​)

Where H(Di)H(D_i)H(Di​) is the entropy of the subset DiD_iDi​. It represents the “remaining uncertainty” after we have split the data using feature AAA.

Then we can give the definition of Information Gain:

Definition1.4 :Information Gain represents the reduction in entropy (uncertainty) achieved by partitioning the dataset according to feature AAA.

g(D,A)=H(D)−H(D∣A)g(D, A) = H(D) - H(D|A) g(D,A)=H(D)−H(D∣A)

We want to choose the feature AAA that maximizes g(D,A)g(D, A)g(D,A). A high information gain means that knowing feature AAA clears up a lot of confusion about the class labels.That is the core idea of next algorithms.

Algorithms

ID3 algorithm

The algorithm starts at the root node. At each node, it calculates the Information Gain g(D,A)g(D, A)g(D,A) for every available feature AAA. It selects the feature that reduces the uncertainty of the class labels the most.

  • If all instances in DDD belong to the same class CkC_kCk​, the node becomes a leaf labeled CkC_kCk​.
  • If the feature set is empty, the node becomes a leaf labeled with the most frequent class in DDD.
  • Calculate g(D,A)g(D, A)g(D,A) for each feature and select Ag=arg⁡max⁡Ag(D,A)A_g = \arg\max_A g(D, A)Ag​=argmaxA​g(D,A).

In recursive steps:

  • For each possible value aia_iai​ of feature AgA_gAg​, create a branch and a new child node.
  • Divide DDD into subsets DiD_iDi​ based on Ag=aiA_g = a_iAg​=ai​.
  • Repeat the process for each DiD_iDi​.

CART algorithm

Unlike ID3 or C4.5, CART always produces a binary tree.

CART Regression Trees

A regression tree can be viewed as a model that partitions the feature space into R1,R2,…,RMR_1, R_2, \dots, R_MR1​,R2​,…,RM​ regions. The model’s prediction for any input xxx falling into region RmR_mRm​ is a constant cmc_mcm​:

f(x)=∑m=1McmI(x∈Rm)f(x) = \sum_{m=1}^{M} c_m I(x \in R_m) f(x)=m=1∑M​cm​I(x∈Rm​)

Where III is the indicator function.

Suppose we partition the space into two regions R1(j,s)R_1(j, s)R1​(j,s) and R2(j,s)R_2(j, s)R2​(j,s) based on feature jjj and split point sss:

R1(j,s)={x∣x(j)≤s},R2(j,s)={x∣x(j)>s}R_1(j, s) = \{x | x^{(j)} \le s\}, \quad R_2(j, s) = \{x | x^{(j)} > s\} R1​(j,s)={x∣x(j)≤s},R2​(j,s)={x∣x(j)>s}

The goal is to solve:

min⁡j,s[∑xi∈R1(j,s)(yi−c^1)2+∑xi∈R2(j,s)(yi−c^2)2]\min_{j, s} \left[ \sum_{x_i \in R_1(j, s)} (y_i - \hat{c}_1)^2 + \sum_{x_i \in R_2(j, s)} (y_i - \hat{c}_2)^2 \right] j,smin​​xi​∈R1​(j,s)∑​(yi​−c^1​)2+xi​∈R2​(j,s)∑​(yi​−c^2​)2​

The algorithm steps:

  • For each feature jjj, consider all possible split points sss.
  • For each pair (j,s)(j, s)(j,s), calculate the resulting sum of squared errors in the two new branches and select the pair that yields the minimum total loss function.
  • Recursively apply this to the resulting sub-regions until a stopping criterion (e.g., max depth or minimum samples) is met.

CART Classification Trees

If there are KKK classes, and the probability of a sample belonging to class kkk is pkp_kpk​, the Gini Index of the probability distribution is:

Gini(p)=∑k=1Kpk(1−pk)=1−∑k=1Kpk2Gini(p) = \sum_{k=1}^K p_k(1-p_k) = 1 - \sum_{k=1}^K p_k^2 Gini(p)=k=1∑K​pk​(1−pk​)=1−k=1∑K​pk2​

For a dataset DDD, the
Gini Index is:

D1={(x,y)∈D∣x(A)=a};D2={(x,y)∈D∣x(A)≠a}D_1 = \{(x, y) \in D \mid x^{(A)} = a\}; D_2 = \{(x, y) \in D \mid x^{(A)} \neq a\}D1​={(x,y)∈D∣x(A)=a};D2​={(x,y)∈D∣x(A)=a}

The algorithm steps:

  • For every available feature AAA, and for every possible value aaa that AAA can take, calculate Gini(D,A)Gini(D, A)Gini(D,A).

  • Choose the feature A∗A^*A∗ and the split point a∗a^*a∗ that result in the minimum Gini Index:

(A∗,a∗)=arg⁡min⁡A,aGini(D,A)(A^*, a^*) = \arg\min_{A, a} Gini(D, A) (A∗,a∗)=argA,amin​Gini(D,A)

Recursive Branching:

  • Create two child nodes and distribute the samples into D1D_1D1​ and D2D_2D2​.

  • Repeat the process for each child node.

Stop splitting if and only if:

  • The number of samples in a node is less than a threshold.

  • The Gini Index is below a threshold (the node is pure enough).

  • The tree reaches a maximum depth.
