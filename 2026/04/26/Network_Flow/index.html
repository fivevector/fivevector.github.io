Network Flow

Definitions

Definition 1 (Flow Network) :
A flow network is a directed graph G=(V,E)G = (V, E)G=(V,E) together with two distinguished vertices: a source sss and a sink ttt. Each edge (u,v)∈E(u, v) \in E(u,v)∈E is assigned a nonnegative capacity c(u,v)≥0c(u, v) \ge 0c(u,v)≥0. If (u,v)∉E(u, v) \notin E(u,v)∈/E, we set c(u,v)=0c(u, v) = 0c(u,v)=0.

We assume without loss of generality that no edge enters sss and no edge leaves ttt, and that every vertex lies on some path from sss to ttt.

Definition 2 (Net Flow) :

  1. Capacity constraint:

    f(u,v)≤c(u,v).f(u, v) \le c(u, v). f(u,v)≤c(u,v).

  2. Skew symmetry:

    f(u,v)=−f(v,u).f(u, v) = -f(v, u). f(u,v)=−f(v,u).

  3. Flow conservation: For every vertex u∈V∖{s,t}u \in V \setminus \{s, t\}u∈V∖{s,t},

    ∑v∈Vf(u,v)=0.\sum_{v \in V} f(u, v) = 0. v∈V∑​f(u,v)=0.

The Max-Flow Min-Cut Theorem

This part is the central theorem of network flow theory.

Theorem 3 (Max-Flow Min-Cut) :
The following three statements are equivalent:

  1. There exists a cut (S,T)(S, T)(S,T) such that ∣f∣=c(S,T)|f| = c(S, T)∣f∣=c(S,T).
  2. fff is a maximum flow.
  3. fff admits no augmenting paths (i.e., there is no path from sss to ttt in GfG_fGf​).

(1) ⇒\Rightarrow⇒ (2):

The Ford‑Fulkerson Algorithm

The proof of the Max‑Flow Min‑Cut Theorem directly suggests an algorithm: start with the zero flow and repeatedly augment along any sss-ttt path in the residual graph until no such path exists. This is the Ford‑Fulkerson method.

If capacities are integers, the flow increases by at least 1 in each augmentation, so the algorithm terminates. However, when capacities are irrational or large, the number of iterations can be enormous—even with only 4 vertices, a poor choice of augmenting paths can require billions of augmentations on a graph with capacities 10910^9109 and 1.

The Edmonds‑Karp Algorithm

Edmonds and Karp observed that always choosing a shortest augmenting path (with edge weights 1) dramatically improves the worst case. This is equivalent to using breadth‑first search (BFS) to find the augmenting path.

Analysis (sketch).

  • A BFS takes O(E)O(E)O(E) time.
  • With this choice, each edge can serve as the “critical” edge (the one determining cf(p)c_f(p)cf​(p)) at most O(V)O(V)O(V) times, leading to at most O(VE)O(VE)O(VE) augmentations.
  • Overall complexity: O(VE2)O(VE^2)O(VE2).

This was the first polynomial‑time algorithm for maximum flow. Later developments have led to even faster algorithms (e.g., King–Rao–Tarjan O(VElog⁡E/(Vlog⁡V)V)O(VE \log_{E/(V \log V)} V)O(VElogE/(VlogV)​V), and Orlin’s recent O(VE)O(VE)O(VE) algorithm).
